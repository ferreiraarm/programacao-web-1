Introdução

Ele é um "framework" (um conjunto de regras e ferramentas) que ajuda os desenvolvedores a criar aplicativos da web sem ter que começar do zero.

  1. Componentes: Os componentes são como peças de um quebra-cabeça. Eles dividem a interface do usuário em partes menores e reutilizáveis. Cada componente tem sua própria lógica e aparência, facilitando a construção e manutenção de aplicativos complexos.

  2. Templates: Os templates são como moldes que dizem ao Angular como exibir os componentes na tela. Eles usam uma linguagem especial para combinar lógica e elementos visuais, permitindo criar a aparência do aplicativo de forma dinâmica.

  3. Serviços: Os serviços são como ajudantes que fornecem funcionalidades compartilháveis. Eles podem lidar com tarefas como buscar dados da internet, processar informações ou gerenciar estados. Os componentes podem usar serviços para realizar essas tarefas sem repetir código.

  4. Roteamento: O roteamento é como um mapa que direciona os usuários para diferentes partes do aplicativo. Permite criar páginas diferentes e controlar qual página é exibida quando os usuários interagem com botões ou links.

  5. Módulos: Os módulos são como caixas organizadoras. Eles agrupam componentes, serviços e outros recursos relacionados em uma estrutura organizada. Isso ajuda a dividir o aplicativo em partes gerenciáveis e a evitar confusão.

  6. Injeção de Dependência: Isso é como fornecer peças necessárias para construir algo. O Angular cuida de fornecer automaticamente os serviços e outros recursos necessários aos componentes, garantindo que tudo funcione sem que você precise se preocupar com a configuração detalhada.

Versões
  1. AngularJS (versão 1.x): Lançado em 2010, o AngularJS foi a primeira versão do framework. Ele trouxe a ideia de usar diretivas HTML para criar interatividade e dinamismo em páginas web. No entanto, ele tinha algumas limitações de desempenho e arquitetura.

  2. Angular 2+: Lançado em 2016, o Angular 2 foi uma reescrita completa do AngularJS. Foi uma mudança radical, o que levou à mudança no nome para simplesmente "Angular". As versões subsequentes, como Angular 4, 5, 6, e assim por diante, trouxeram melhorias incrementais e novos recursos.

  3. Angular 2.0+: A partir do Angular 2, o framework foi reescrito em TypeScript (uma linguagem de programação baseada em JavaScript) e introduziu o conceito de componentes, que se tornou o bloco de construção fundamental de aplicativos Angular. Além disso, foi aprimorado para ter melhor desempenho, modularidade e escalabilidade.
  4. Angular 4+: As versões posteriores do Angular 2 foram numeradas sequencialmente, pulando a versão 3 por razões técnicas. As versões 4, 5, 6 e assim por diante trouxeram melhorias na performance, otimizações de tamanho, recursos aprimorados e uma experiência de desenvolvimento mais eficiente.

  5. Angular 7, 8, 9: Essas versões continuaram a melhorar a performance, introduziram recursos como Angular CLI (uma interface de linha de comando para desenvolvimento Angular) e expandiram as capacidades do Angular Material, uma biblioteca de componentes de interface de usuário.

  6. Angular 10, 11, 12: Essas versões continuaram aprimorando a performance e trazendo melhorias nas ferramentas e na experiência de desenvolvimento. Também houve atualizações nas bibliotecas e recursos oferecidos pelo Angular.
